1. 개요

1. Overview

LightHarness는 Claude Pro 요금제 안에서 소규모·중규모 작업을 효율적으로 처리하는 경량 에이전트 팀 프레임워크다. 3-Tier 라우팅 위에 올라가는 워크플로우 오케스트레이터로, 슬래시 커맨드로 동작한다.

LightHarness is a lightweight agent team framework for efficiently handling small-to-medium tasks within Claude Pro. It acts as a workflow orchestrator on top of 3-Tier routing, operated via slash commands.

핵심 설계 원칙은 cloud(Claude 직접) 사용을 최소화하고 판단·조율만 맡기며, 실작업은 Tier 2/3에 위임하는 것이다. cloud 스텝은 세션당 10개 이하를 목표로 한다.

The core principle is to minimize direct cloud (Claude) usage — only judgment and coordination — while delegating actual work to Tier 2/3. Target: ≤10 cloud steps per session.

6. Queue 머지 모드

6. Queue Merge Mode

할 일 목록에 쌓인 Blueprint들을 /harness-run 한 번에 효율적으로 처리한다. cloud 단계를 배치 내에서 1회로 병합해 실행 횟수를 절감하는 것이 핵심이다.

Processes multiple queued Blueprints in a single /harness-run call. The key is merging cloud steps within a batch into one execution to reduce API calls.

8. Blueprint Lint — 설계 직후 자동 실행

8. Blueprint Lint — Auto-runs After Design

/harness 로 Blueprint를 설계하면 Lint가 즉시 실행되어 아래 5가지 규칙을 검사한다. 위반 시 실행이 차단된다.

When you design a Blueprint with /harness, Lint runs immediately and checks 5 rules. Violations block execution.

  • RULE-1 Workflow의 To 값이 Agent Id 또는 ENDWorkflow To values match an Agent Id or END
  • RULE-2 local 에이전트 프롬프트에 한글 없음No Korean characters in local agent prompts
  • RULE-3 cloud 비율 ≤ 0.5Cloud agent ratio ≤ 0.5
  • RULE-4 모든 Workflow step에 Error_Policy 있음Every Workflow step has Error_Policy
  • RULE-5 Queue.affected_files 비어있지 않음Queue.affected_files is not empty
